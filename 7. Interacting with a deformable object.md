I’ll explain this tutorial **in simple steps**, just like the previous ones (rigid object and articulation). The main new idea here is **deformable objects**, which behave very differently from rigid objects.

---

# 1. What is a **deformable object**?

A deformable object is something that **changes shape when forces are applied**.

Examples in the real world:

* sponge
* jelly
* rubber cube
* soft foam
* human tissue

Rigid objects (previous tutorial):

```
shape never changes
```

Deformable objects:

```
shape can bend, stretch, squash
```

Example concept:

```
Rigid cube         Soft cube

████               ████
████    → push →   █  █
████               ███
```

---

# 2. PhysX soft body simulation

In Isaac Lab (PhysX), deformable objects are called:

```
soft bodies
```

These are simulated using:

```
FEM (Finite Element Method)
```

FEM divides the object into **many small elements** that can deform.

---

# 3. Two meshes inside a soft body

A deformable object actually has **two meshes**.

### 1️⃣ Simulation mesh

Used to compute deformation physics.

Example internal structure:

```
 tetrahedrons inside cube
```

### 2️⃣ Collision mesh

Used for detecting collisions with:

* ground
* other objects

Conceptual structure:

```
         Collision mesh
        ┌─────────────┐
        │             │
        │ Simulation  │
        │    mesh     │
        │             │
        └─────────────┘
```

Simulation mesh = physics
Collision mesh = contacts

---

# 4. What this tutorial does

The tutorial:

1️⃣ spawns **soft cubes**
2️⃣ resets their state
3️⃣ moves some mesh vertices
4️⃣ runs the simulation

Result:

```
some cubes fall
some cubes move upward
```

# 5. Important limitation

⚠️ Deformable objects only work on:

```
GPU simulation
```

They are **too expensive for CPU** physics.

# 7. Creating the deformable cube

```python
cfg = DeformableObjectCfg(...)
```

This configuration describes the cube.

---

### Cube size

```python
size=(0.2, 0.2, 0.2)
```

So the cube is:

```
20 cm × 20 cm × 20 cm
```

---

### Deformable physics properties

```python
DeformableBodyPropertiesCfg(...)
```

These control:

* contact behavior
* simulation stability

Example parameters:

| Parameter      | Meaning                 |
| -------------- | ----------------------- |
| rest_offset    | distance before contact |
| contact_offset | collision tolerance     |

---

### Material properties

```python
DeformableBodyMaterialCfg(
 poisson_ratio=0.4,
 youngs_modulus=1e5
)
```

These determine **how soft or stiff the object is**.

Important parameters:

### Poisson's ratio

```
how much object expands sideways
```

Example:

```
press sponge
→ sides expand
```

---

### Young's modulus

Controls stiffness.

```
small value → soft object
large value → stiff object
```

Example scale:

```
jelly → low
rubber → medium
steel → very high
```

# 10. Important difference from rigid bodies

Rigid bodies store state like this:

```
position
rotation
velocity
```

Deformable bodies store state differently.

Their state is defined by:

```
positions of ALL mesh nodes
```

Each cube might have:

```
hundreds of vertices
```

Each vertex has:

```
x y z position
velocity
```

---

# 11. Resetting the deformable object

Get the default nodal state:

```python
nodal_state = cube_object.data.default_nodal_state_w.clone()
```

This contains:

```
all vertex positions
all vertex velocities
```

---

# 12. Randomizing object pose

The code generates random position:

```python
pos_w = torch.rand(...) * 0.1 + origins
```

Meaning:

```
random small offset
```

So cubes spawn slightly differently every reset.

---

### Random orientation

```python
quat_w = random_orientation(...)
```

So cubes rotate randomly.

---

### Transform nodes

```python
transform_nodal_pos(...)
```

This moves **all vertices** of the cube to the new position.

Important:

```
rigid objects move one pose
deformable objects move all vertices
```

---

# 13. Writing nodal state

```python
cube_object.write_nodal_state_to_sim(nodal_state)
```

This sends the **vertex positions and velocities** to the simulator.

---

# 14. Kinematic targets

Now we introduce a new idea:

```
kinematic node control
```

This means:

```
user manually controls some vertices
```

while the rest are simulated.

Example idea:

```
pull one corner of jelly
→ rest of jelly stretches
```

---

### Kinematic target structure

Each node has:

```
[x y z position, flag]
```

Flag:

```
0 → kinematic (controlled)
1 → free (simulated)
```

---

### Free all vertices initially

```python
nodal_kinematic_target[...,3] = 1.0
```

Meaning:

```
all nodes free
```

---

# 15. Moving some nodes

Later in the simulation:

```python
nodal_kinematic_target[[0,3],0,2] += 0.001
```

Meaning:

```
cubes 0 and 3
node index 0
increase z position
```

So one corner of the cube moves upward.

---

### Constrain the node

```python
nodal_kinematic_target[[0,3],0,3] = 0.0
```

Meaning:

```
node becomes kinematic
```

So that vertex **follows the target exactly**.

---

# 16. Resulting motion

Because one corner moves up:

```
cube stretches upward
```

Visualization:

```

  ████

after

  ████
  █  █
   ██
```

# 20. Printing cube position

Deformable objects do not have a single root.

So Isaac Lab approximates the root as:

```

```

Example:

```python

```

# 22. Big picture comparison

| Feature     | Rigid Object | Articulation  | Deformable     |
| ----------- | ------------ | ------------- | -------------- |
| Bodies      | 1            | many          | mesh           |
| Joints      | ❌            | ✅             | ❌              |
| Deformation | ❌            | ❌             | ✅              |
| State       | pose         | pose + joints | node positions |
| Control     | forces       | torques       | node targets   |

---

✅ **Simple summary**

This tutorial shows how to simulate soft objects (deformable cubes).
Instead of controlling a rigid body pose, we control **individual mesh nodes**, allowing the object to **bend, stretch, and deform** using FEM physics.
