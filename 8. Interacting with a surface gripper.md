I’ll explain this **in a very simple way**, step-by-step. Think of this tutorial as **teaching a robot how to open and close a suction gripper in simulation**.

---

# 1. What this tutorial is about

This tutorial shows how to control a **surface gripper** attached to a robot in **Isaac Lab / Isaac Sim**.

A **surface gripper** is like a **vacuum suction cup** used in factories.

Example:
Robots picking boxes with suction.

Example workflow:

```
Robot moves → suction touches object → suction activates → object sticks → robot moves object
```

⚠ Important note in the tutorial:

> Surface grippers currently work **only on CPU**, not GPU.

So you must run:

```bash
./isaaclab.sh -p scripts/tutorials/01_assets/run_surface_gripper.py --device=cpu
```

---

# 2. Big picture of the code

The script does **4 main things**:

1️⃣ Start Isaac Sim
2️⃣ Create the scene (floor, lights, robots)
3️⃣ Add a **surface gripper** to the robot
4️⃣ Run a simulation loop where the gripper randomly opens or closes

Simplified flow:

```
Start simulator
      ↓
Create environment
      ↓
Spawn robots + grippers
      ↓
Simulation loop:
    reset robot
    send command to gripper
    step simulation
    read gripper state
```

```python
cfg = sim_utils.DomeLightCfg(...)
```

```python
origins = [[2.75,0,0], [-2.75,0,0]]
```

Two robots will appear here:

```
World
```

# 5. Creating the robot

```python
pick_and_place_robot_cfg = PICK_AND_PLACE_CFG.copy()
pick_and_place_robot_cfg.prim_path = "/World/Origin.*/Robot"
```

This loads a **predefined robot configuration**.

The robot is a **simple pick-and-place robot**.

It can move:

```
X direction
Y direction
Z direction
```

Like a **3-axis gantry robot** used in factories.

---

Then it creates the robot:

```python
pick_and_place_robot = Articulation(cfg=pick_and_place_robot_cfg)
```

Important concept:

**Articulation = a robot with joints**

Examples:

```
Robot arm
Humanoid
Mobile manipulator
```

# 6. Creating the surface gripper

Now we create the **vacuum gripper**.

First create configuration:

```python
surface_gripper_cfg = SurfaceGripperCfg()
```

Then tell it **where the gripper is located on the robot**.

```python
surface_gripper_cfg.prim_path =
"/World/Origin.*/Robot/picker_head/SurfaceGripper"
```

Meaning:

```
Robot
 └── picker_head
       └── SurfaceGripper
```

---

# 7. Important gripper parameters

### 1️⃣ Max grip distance

```python
max_grip_distance = 0.1
```

Meaning:

```
object must be within 10 cm
to be grabbed
```

---

### 2️⃣ Shear force limit

```python
shear_force_limit = 500
```

Force **sideways**.

Example:

```
object sliding sideways
```

---

### 3️⃣ Coaxial force limit

```python
coaxial_force_limit = 500
```

Force **along suction direction**.

Example:

```
object pulling away
```

---

### 4️⃣ Retry interval

```python
retry_interval = 0.1
```

If gripping fails, the gripper tries again every **0.1 seconds**.

---

Then the gripper is created:

```python
surface_gripper = SurfaceGripper(cfg=surface_gripper_cfg)
```

Now the scene contains:

```
World
 ├── Robot1
 │     └── SurfaceGripper
 └── Robot2
       └── SurfaceGripper
```

# 8. Running the simulation

Function:

```
run_simulator()
```

This is the **main simulation loop**.

---

# 9. Resetting the robot

Every 500 steps:

```python
if count % 500 == 0:
```

The simulation resets.

Why?

Because many robotics simulations **periodically reset environments**.

Common in:

```
Reinforcement Learning
Robot testing
```

---

Reset robot position:

```python
robot.write_root_pose_to_sim(...)
```

Reset joint positions:

```python
robot.write_joint_state_to_sim(...)
```

---

Reset gripper:

```python
surface_gripper.reset()
```

This means:

```
open gripper
clear buffers
reset state
```

---

# 10. Sending commands to the gripper

Each step the code generates a random command.

```python
gripper_commands = torch.rand(...) * 2 - 1
```

Range:

```
-1 → 1
```

---

### Command interpretation

```
-1   → Opening
-0.3 → Idle
0.3  → Closing
1
```

Example:

| Command | Behavior |
| ------- | -------- |
| -0.8    | Opening  |
| 0.0     | Idle     |
| 0.9     | Closing  |

---

Send command:

```python
surface_gripper.set_grippers_command(gripper_commands)
```

---

Then send data to physics engine:

```python
surface_gripper.write_data_to_sim()
```

Meaning:

```
Python → PhysX physics engine
```

---

# 11. Stepping the simulation

```python
sim.step()
```

This runs **one physics step**.

Think of it like:

```
time += dt
update physics
move objects
apply forces
```

---

# 12. Reading the gripper state

After stepping the simulation:

```python
surface_gripper.update(sim_dt)
```

This updates internal buffers.

---

Then read the state:

```python
surface_gripper_state = surface_gripper.state
```

Possible values:

| Value | Meaning |
| ----- | ------- |
| -1    | Open    |
| 0     | Closing |
| 1     | Closed  |

Example output:

```
Gripper state: tensor([-1,1])
Mapped commands: ['Open','Closed']
```

# 14. Important concept: InteractiveScene

In real projects you usually use:

```
InteractiveScene
```

Example:

```python
scene = InteractiveScene()
scene.surface_grippers["gripper"] = surface_gripper
```

This automatically handles:

```
write_data_to_sim()
update()
```

So your code becomes simpler.
