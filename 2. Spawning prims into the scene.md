This tutorial teaches **how to add objects into the Isaac Sim world using Python**.

---

# 1. What this tutorial is about

In the previous tutorial you created an **empty simulation world**.

Now you will **add objects into that world**.

These objects are called:

```
Prims
```

Examples of prims:

* ground plane
* lights
* shapes (cones, cubes)
* meshes (tables, robots)
* cameras

So the goal is:

```
Empty scene → add objects → run simulation
```

---

# 2. What is a “Prim”?

In **USD (Universal Scene Description)**, everything in the scene is a **Prim**.

Think of a prim like a **node in a scene tree**.

Example scene structure:

```
/World
   ├── GroundPlane
   ├── Light
   └── Objects
        ├── Cone1
        ├── Cone2
        ├── Table
```

Each item is a **prim**.

### 1️⃣ Prims

Objects in the scene.

Example:

```
Cone

### 2️⃣ Attributes

Properties of objects.

Example:

```
color = red

```

### 3️⃣ Relationships

Connections between objects.

Example:

```
mesh → material

### 4️⃣ USD Stage

The **entire scene**.

Think of it like:

```
USD Stage = the whole 3D world
```

# 4. How Isaac Lab spawns objects

Isaac Lab simplifies object creation using **configuration classes**.

The workflow looks like this:

### Step 1 — Create configuration

Example:

```python
cfg = ConeCfg(...)
```

This defines:

```
size
color
physics properties
```

---

### Step 2 — Spawn the object

Example:

```python
cfg.func("/World/Cone", cfg)
```

This **places the object into the scene**.

---

# 5. Important rule

⚠️ All objects must be created **before the simulation starts**.

Good workflow:

```
1 Create objects
2 Start simulation
3 Run physics
```

Bad workflow:

```
Start simulation
Add objects later
```

This can break **GPU physics buffers**.

---

# 6. Step 1 — Spawn a ground plane

First we add the **floor**.

```python
cfg_ground = GroundPlaneCfg()
cfg_ground.func("/World/defaultGroundPlane", cfg_ground)
```

This creates a **grid floor** so objects don't fall forever.

---

# 7. Step 2 — Add a light

Next we add lighting so objects are visible.

Example:

```python
cfg_light_distant = DistantLightCfg(...)
```

This is like **sunlight**.

Characteristics:

```
very far away
light comes from one direction
```

We place it above the scene:

```
position = (1,0,10)
```

---

# 8. Step 3 — Create a group for objects

Before adding shapes, we create a **container node**.

```python
create_prim("/World/Objects", "Xform")
```

`Xform` means **transform node**.

Purpose:

```
group objects together
move them as a group
organize the scene
```

Now objects go inside:

```
/World/Objects/Cone1
/World/Objects/Cone2
```

---

# 9. Step 4 — Spawn simple cones

We create a **red cone**.

```python
cfg_cone = ConeCfg(...)
```

Properties:

```
radius
height
color
```

Then we place two cones:

```
Cone1
Cone2
```

Positions:

```
(-1, 1, 1)
(-1,-1, 1)
```

Important:

These cones are **visual only**.

They have:

```
no physics
no collisions
```

So they will **not move**.

---

# 10. Step 5 — Spawn a cone with physics

Next we create a **green cone that can fall**.

Extra properties added:

```
RigidBodyProperties
MassProperties
CollisionProperties
```

Meaning:

```
gravity affects it
it has weight
it collides with objects
```

So this cone will:

```
fall down
hit the table
hit the ground
```

---

# 11. Step 6 — Spawn a deformable object

Next we create a **blue cuboid** with **soft-body physics**.

```python
MeshCuboidCfg(...)
```

This adds:

```
DeformableBodyProperties
```

Meaning:

```
object can bend or deform
```

Example materials:

```
rubber
jello
cloth
```

Important:

Soft bodies only work on **GPU simulation**.

---

# 12. Step 7 — Spawn a table from a file

Instead of creating shapes manually, we can **load objects from files**.

Example:

```python
UsdFileCfg(...)
```

This loads a **table model** stored in a USD file.

The path:

```
ISAAC_NUCLEUS_DIR/Props/Mounts/SeattleLabTable
```

This file already contains:

```
3D mesh
materials
textures
```

So the table appears instantly.

---

# 13. What is “reference” loading?

When the table is loaded, it is **not copied into the scene**.

Instead, the scene stores a **reference**.

Think of it like:

```
Scene → pointer → table file
```

Benefits:

```
smaller scene
faster loading
non-destructive editing
```

Example:

You can change the **table color** in the scene without modifying the original asset file.

# 14. Running the simulation

The main function does:

### Step 1 — create simulation

```
SimulationContext
```

---

### Step 2 — set camera

```
sim.set_camera_view()
```

---

### Step 3 — build scene

```
design_scene()
```

This adds:

```
ground
light
cones
table
```

---

### Step 4 — start simulation

```
sim.reset()
```

---

### Step 5 — simulation loop

```
while simulation_app.is_running():
    sim.step()
```

Each step updates:

```
physics
collisions
rendering
```
