We will break it into **5 simple ideas**:

1. What an RL environment is
2. What is different from the Base Environment
3. Rewards (how the agent knows if it did well)
4. Termination (when an episode stops)
5. How everything runs in the simulation loop

# 1. What is a Manager-Based RL Environment?

In the previous tutorial you created a **Base Environment**.

That environment was simply:

```
Agent → send action → Environment
Environment → send observation → Agent
```

Example:

Agent sends:

```
move cart left/right
```

Environment returns:

```
pole angle
cart velocity
```

This is enough for **robot control**, but **not enough for learning**.

Why?

Because the agent **does not know if it is doing well or badly**.

So for **Reinforcement Learning (RL)** we must add:

* **Reward** → how good the action was
* **Termination** → when the episode ends
* **Commands** → optional goals
* **Curriculum** → gradually harder tasks

So the RL environment becomes:

```
Agent → action
Environment → observation + reward + done
```

---

# 2. Base Environment vs RL Environment

### Base Environment

Simple interaction:

```
obs = env.step(action)
```

Returns:

```
observation
```

Used for:

* traditional robotics
* motion control
* planning

---

### RL Environment

More signals are returned:

```
obs, reward, terminated, truncated, info = env.step(action)
```

Meaning:

| Output      | Meaning                          |
| ----------- | -------------------------------- |
| observation | what the robot sees              |
| reward      | how good the action was          |
| terminated  | episode ended because of failure |
| truncated   | episode ended because of time    |
| info        | debug information                |

---

# 3. Rewards (How the Agent Learns)

In RL the **reward tells the agent what behavior is good**.

Cartpole task:

Goal:

```
keep the pole upright
```

So rewards are defined like this.

---

### Reward 1: Alive Reward

```
alive = RewTerm(func=mdp.is_alive, weight=1.0)
```

Meaning:

```
+1 reward every step if the system is still running
```

Encourages:

```
stay alive longer
```

---

### Reward 2: Termination Penalty

```
terminating = RewTerm(func=mdp.is_terminated, weight=-2.0)
```

Meaning:

```
-2 reward if episode fails
```

Encourages:

```
avoid failure
```

---

### Reward 3: Keep Pole Upright

```
pole_pos = RewTerm(
    func=mdp.joint_pos_target_l2,
    weight=-1.0,
)
```

Meaning:

```
reward is higher when pole angle ≈ 0
```

Because upright pole = angle 0.

If pole falls:

```
reward becomes worse
```

---

### Reward 4: Small Cart Velocity

```
cart_vel
```

Encourages:

```
cart not moving too fast
```

---

### Reward 5: Small Pole Velocity

```
pole_vel
```

Encourages:

```
pole not swinging too much
```

---

### Final reward idea

Total reward is:

```
reward =
    alive
  + pole_pos
  + cart_vel
  + pole_vel
  + termination_penalty
```

The agent learns to **maximize total reward**.

---

# 4. Termination Conditions (When Episode Ends)

Episodes cannot run forever.

They end when something happens.

Two cases:

---

### 1. Time Limit

```
time_out = DoneTerm(func=mdp.time_out)
```

Meaning:

```
episode ends after max time
```

Example:

```
5 seconds
```

---

### 2. Cart Out of Bounds

```
cart_out_of_bounds
```

Meaning:

```
if cart position < -3 or > 3
episode ends
```

Because robot went too far.

---

### Example episode

```
Start episode
pole upright

step 1
step 2
step 3
...

pole falls
→ terminate episode
→ reset environment
```

---

# 5. Commands (Optional Goals)

Some tasks require **targets**.

Examples:

Navigation robot:

```
goal position = (x,y)
```

Quadruped locomotion:

```
target speed = 1.5 m/s
```

Manipulator:

```
target object position
```

These goals are handled by:

```
CommandManager
```

But **cartpole does not need commands**, so it is set to:

```
None
```

---

# 6. Curriculum Learning (Optional)

Curriculum means:

```
start easy → gradually harder
```

Example:

Cartpole training progression:

Step 1

```
pole starts almost upright
```

Step 2

```
pole starts more tilted
```

Step 3

```
stronger disturbances
```

This helps RL learn faster.

In this tutorial:

```
curriculum = None
```

# 8. Running the Simulation Loop

The simulation loop:

```
while simulator running:
```

---

### Step 1 — Reset

Every 300 steps:

```
env.reset()
```

This:

* resets robot
* resets pole
* resets episode

---

### Step 2 — Sample Action

Random action for demo:

```
joint_efforts = torch.randn(...)
```

In real RL:

```
action = policy(observation)
```

---

### Step 3 — Step Environment

```
obs, rew, terminated, truncated, info = env.step(action)
```

Environment does:

1️⃣ apply action to robot
2️⃣ simulate physics
3️⃣ compute observation
4️⃣ compute reward
5️⃣ check termination

---

### Step 4 — Print Observation

```
print pole angle
```

# 11. Why Isaac Lab Uses Managers

Managers organize complexity.

Instead of one huge environment file:

```
ManagerBasedRLEnv
│
├── SceneManager
├── ActionManager
├── ObservationManager
├── EventManager
├── RewardManager
└── TerminationManager
```

This makes it **modular and scalable**.
