I’ll explain this tutorial **very simply**, because this part of Isaac Lab can be confusing at first. The key idea is:

> **The environment defines the task, while the agent configuration defines how the RL algorithm trains.**

So this tutorial is about **how to choose and configure different RL libraries and algorithms**.

---

# 1. Big Picture: Two Things in RL Training

When training a robot in Isaac Lab, you need **two configurations**:

### 1️⃣ Environment configuration

Defines the **task**.

Example:

* Cartpole balancing
* Robot walking
* Drone flying

Example config:

```text
CartpoleEnvCfg
```

It defines:

* physics
* reward
* observation
* termination
* robot model

---

### 2️⃣ Agent configuration

Defines **how the RL algorithm learns**.

Example:

* PPO
* SAC
* TD3

It defines things like:

```text
learning rate
batch size
network size
number of steps
discount factor
```

---

# 2. Why Isaac Lab Supports Many RL Libraries

Different research groups prefer different RL libraries.

Common ones supported by Isaac Lab:

| Library               | Description        |
| --------------------- | ------------------ |
| **Stable-Baselines3** | popular RL library |
| **RSL-RL**            | very fast GPU RL   |
| **RL-Games**          | used by NVIDIA     |
| **SKRL**              | modern RL library  |

Each library needs **its own configuration**.

# 4. How Isaac Lab Connects Environment and RL Library

This happens inside the **environment registration**.

Example:

```python
gym.register(
    id="Isaac-Cartpole-v0",
    entry_point="isaaclab.envs:ManagerBasedRLEnv",
    kwargs={
        "env_cfg_entry_point": "...CartpoleEnvCfg",
        "rl_games_cfg_entry_point": "...rl_games_ppo_cfg.yaml",
        "rsl_rl_cfg_entry_point": "...CartpolePPORunnerCfg",
        "skrl_cfg_entry_point": "...skrl_ppo_cfg.yaml",
        "sb3_cfg_entry_point": "...sb3_ppo_cfg.yaml",
    },
)
```

This registers **multiple training configurations** for the same task.

Think of it like this:

```text
Task: Cartpole

Training options:
    Stable-Baselines3 PPO
    RSL-RL PPO
    RL-Games PPO
    SKRL PPO
```

---

# 5. What These Keys Mean

Inside `kwargs`:

```python
"sb3_cfg_entry_point"
```

means:

```text
Stable-Baselines3 configuration
```

---

Example:

```python
"sb3_cfg_entry_point": "...sb3_ppo_cfg.yaml"
```

This file contains parameters like:

```yaml
learning_rate: 0.0003
gamma: 0.99
n_steps: 2048
batch_size: 64
```

These parameters define **how PPO trains**.

---

Another example:

```python
"rsl_rl_cfg_entry_point": "...CartpolePPORunnerCfg"
```

This uses a **Python class configuration** instead of YAML.

---

# 6. Why They Use Strings Instead of Importing Classes

Example:

Preferred way:

```python
"rsl_rl_cfg_entry_point": "agents.rsl_rl_ppo_cfg:CartpolePPORunnerCfg"
```

Alternative (not recommended):

```python
from agents.rsl_rl_ppo_cfg import CartpolePPORunnerCfg
```

Why?

Because importing classes:

```text
slows down startup time
```

Using strings means:

```text
load only when needed
```

This is called **lazy loading**.

---

# 7. How Training Script Reads the Agent Config

Inside the training script:

```python
--agent sb3_cfg_entry_point
```

This tells the script:

```text
Use Stable-Baselines3 configuration
```

Then the script searches inside:

```text
kwargs dictionary
```

to find:

```text
sb3_cfg_entry_point
```

---

Example process:

```
User runs:
    train.py --agent sb3_cfg_entry_point

Script looks inside registry:

    kwargs["sb3_cfg_entry_point"]

Loads:

    sb3_ppo_cfg.yaml
```

Then the agent is created.

# 10. What Is Symmetry Augmentation?

Example:

Cartpole environment is symmetric.

If the pole falls:

```text
left side
```

or

```text
right side
```

Physics is identical.

So the RL algorithm can **duplicate training data by mirroring states**.

Example:

```
state: pole leaning left
mirror: pole leaning right
```

This helps training:

```text
faster learning
better generalization
```

---

# 11. Hydra Configuration Override

Hydra lets you modify parameters from the command line.

Example:

```bash
agent.algorithm.symmetry_cfg.use_data_augmentation=false
```

Meaning:

```text
disable symmetry augmentation
```

without editing code.

Very useful for experiments.
