# 2. What Is Policy Inference?

During RL there are **two phases**.

### Training

Robot learns by trial and error.

```
state → policy → action → reward → learning
```

Example:

```
robot falls → gets negative reward → improves
```

---

### Inference

Learning is finished.

Now we only **use the trained policy**.

```
state → policy → action
```

No learning.

Just **control the robot**.

---

# 3. What Is a USD Environment?

USD = **Universal Scene Description**

Developed by Pixar.

It describes **3D worlds**.

Examples:

```
warehouse
factory
office
street
```

In Isaac Lab, environments can be:

1️⃣ **Generated terrain**

```
random rocks
random slopes
procedural terrain
```

2️⃣ **USD scene**

```
warehouse.usd
factory.usd
building.usd
```

This tutorial loads:

```
Simple_Warehouse/warehouse.usd
```

# 4. Overall Workflow

The full process is:

```
Step 1: Train policy
Step 2: Export trained policy
Step 3: Load policy
Step 4: Load USD environment

```

First we train the robot.

Command:

```bash
./isaaclab.sh -p scripts/reinforcement_learning/rsl_rl/train.py \
--task Isaac-Velocity-Rough-H1-v0 --headless
```

Task:

```
Isaac-Velocity-Rough-H1-v0
```

Meaning:

```
Robot: H1 humanoid
Goal: walk with target velocity
Terrain: rough terrain
```

---

# 6. After Training: Export Policy

After training finishes, Isaac Lab exports the policy.

Formats:

```
JIT
ONNX
```

Example file:

```
policy.pt
```

This file contains the **neural network**.

---

# 7. What Is JIT?

JIT = **TorchScript Just-In-Time**

It converts a PyTorch model into a **portable file**.

Advantages:

```
can run without training code
faster inference
easy deployment
```

Think of it like:

```
Training code → compiled robot brain
```

./isaaclab.sh -p scripts/tutorials/03_envs/policy_inference_in_usd.py \

# 9. Loading the Policy

Code:

```python
policy = torch.jit.load(file, map_location=args_cli.device)
```

What happens:

```
policy.pt → loaded as neural network
```

Now we can do:

```
action = policy(observation)
```

Exactly like during training.

---

# 10. Creating the Environment

Next:

```python
env_cfg = H1RoughEnvCfg_PLAY()
```

This loads a **special config for inference**.

Why?

Because training config might include:

```
curriculum learning
randomization
thousands of environments
```

But inference only needs:

```
1 robot
1 environment
```

---

# 11. Reduce Number of Environments

Code:

```python
env_cfg.scene.num_envs = 1
```

During training:

```
4096 environments
```

Why?

To collect data faster.

During inference:

```
only 1 robot
```

---

# 12. Disable Curriculum

Code:

```python
env_cfg.curriculum = None
```

Curriculum learning means:

```
start easy
gradually increase difficulty
```

Example:

```
flat ground → small rocks → big rocks
```

During inference:

```
no training
no curriculum
```

---

# 13. Replace Terrain With USD Scene

This is the **most important modification**.

Original training terrain:

```
random terrain generator
```

Now we replace it with:

```python
env_cfg.scene.terrain = TerrainImporterCfg(
    terrain_type="usd",
    usd_path="warehouse.usd"
)
```

Meaning:

```
load warehouse environment
```

Now the robot walks in:

```
3D warehouse scene
```

instead of generated terrain.

---

# 14. Creating the Environment

Code:

```python
env = ManagerBasedRLEnv(cfg=env_cfg)
```

This builds the simulation.

Inside it loads:

```
robot
warehouse
physics
sensors
```

---

# 15. Reset Environment

Code:

```python
obs, _ = env.reset()
```

This:

```
spawns robot
initializes state
returns first observation
```

---

# 16. Running the Simulation Loop

Main loop:

```python
while simulation_app.is_running():
```

This runs **every simulation step**.

---

# 17. Policy Action

Code:

```python
action = policy(obs["policy"])
```

Meaning:

```
robot observation → neural network → action
```

Example:

```
joint angles
joint velocity
body orientation
```

↓

```
neural network
```

↓

```
joint torques
```

---

# 18. Apply Action to Environment

Code:

```python
obs, _, _, _, _ = env.step(action)
```

This performs:

```
apply action
simulate physics
get new observation
```

Then loop repeats.

---

# 19. Why CPU Is Used

Code:

```python
if args_cli.device == "cpu":
    env_cfg.sim.use_fabric = False
```

Explanation:

During training:

```
4096 environments
GPU faster
```

During inference:

```
1 environment
CPU often faster
```

So they switch to CPU.
