# 2. Problem: Isaac Lab Environment ≠ Normal Gym

Even though Isaac Lab follows the **Gym interface**, it is not exactly the same.

Normal Gym environment:

```
observation → numpy array
action → numpy array
```

Example:

```
obs = [0.1, 0.2, -0.05, 0.3]
```

---

But Isaac Lab environment uses **PyTorch tensors**.

Example:

```
obs = tensor([[...], [...], [...], ...])
```

Shape:

```
(num_envs, observation_size)
```

Example with 64 environments:

```
(64, 4)
```

So:

```
64 cartpoles running simultaneously
```

---

# 3. Problem: RL Libraries Expect Different Interfaces

Different RL libraries expect **different environment formats**.

Examples:

### Stable-Baselines3

expects:

```
VecEnv
list of numpy arrays
```

---

### RSL-RL

expects:

```
torch tensors
```

---

### RL-Games

expects:

```
custom GPU tensors
```

---

So Isaac Lab cannot depend on **only one format**.

Solution:

👉 **Use wrappers**

---

# 4. What is an Environment Wrapper?

A wrapper **converts one interface into another**.

Example:

```
Isaac Lab env
      ↓
SB3 wrapper
      ↓
Stable-Baselines3 compatible env
```

Think of it like a **translator**.

The training script basically does this:

```
1 Start Isaac Sim

```

# 12. Stable Baselines Wrapper

Next wrapper:

```python
env = Sb3VecEnvWrapper(env)
```

This converts Isaac Lab environment into:

```
Stable-Baselines VecEnv
```

Meaning:

```
tensor → numpy
batched env → VecEnv format
```

Without this:

```
SB3 cannot use the environment
```

---

# 13. Observation Normalization

Next wrapper:

```python
VecNormalize
```

This normalizes observations and rewards.

Example:

Instead of:

```
position = 3.5
velocity = 0.02
```

Normalize to:

```
position = 0.15
velocity = 0.01
```

Why?

Because neural networks learn better when inputs are **scaled similarly**.

Benefits:

```
faster training
more stable learning
```

# 17. Training the Agent

The main training call:

```python
agent.learn(
    total_timesteps=n_timesteps
)
```

Example:

```
n_timesteps = 10,000,000
```

Training loop:

```
repeat:

    1 observe state
    2 choose action
    3 apply action
    4 receive reward
    5 update neural network
```

Over millions of steps.
