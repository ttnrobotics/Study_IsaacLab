# 2. What is a “rigid object”?

A **rigid object** is something that:

* has **mass**
* has **collision**
* follows **physics**

Examples:

* box
* ball
* cone
* rock

Rigid means:

```text
The object does NOT deform
```

Example:

| Object      | Type       |
| ----------- | ---------- |
| metal box   | rigid      |
| rubber ball | rigid      |
| cloth       | NOT rigid  |
| jelly       | deformable |

# 3. Two main parts of every simulation

The script separates the simulation into **two steps**.

### Step 1 — Design the scene

This part **creates objects**.

Example:

```text

cone objects
```

This happens **before simulation starts**.

---

### Step 2 — Run the simulation

This part:

* steps the physics engine
* moves objects
* updates their states

Example:

```text
object falls
object collides
object moves
```

---

# 4. Important rule in Isaac Lab

You must follow this order:

```text
1. Create objects
2. Reset simulation
3. Run simulation
```

Once simulation starts:

🚫 Do NOT add new physics objects.

You can only **interact with existing objects**.

# 1. The reset condition

```python
if count % 250 == 0:
```

### What this means

`%` is the **modulo operator** (remainder after division).

Example:

```text
10 % 3 = 1
12 % 3 = 0
```

So this condition means:

```text
Every 250 simulation steps
```

Example timeline:

| Step (count) | Reset? |
| ------------ | ------ |
| 0            | YES    |
| 1            | no     |
| 249          | no     |
| 250          | YES    |
| 500          | YES    |

So **every 250 steps**, the objects are reset.

Why?

* prevents simulation from running forever
* useful for **training reinforcement learning**
* lets objects **restart from new random positions**

---

# 2. Reset simulation counters

sim_time = 0.0

sim_time = 0.0 seconds

Resetting it means:

start a new episode

### `count`

Resetting it ensures the next reset happens again after **250 steps**.

# 3. Get the default root state

```python
root_state = cone_object.data.default_root_state.clone()
```

### What is `root_state`?

Each rigid object has a **root state**.

It contains:

```text
position
orientation
linear velocity
angular velocity
```

Typical format:

```text
[x y z  qx qy qz qw  vx vy vz  wx wy wz]
```

Where:

| Part        | Meaning                  |
| ----------- | ------------------------ |
| x y z       | position                 |
| qx qy qz qw | orientation (quaternion) |
| vx vy vz    | linear velocity          |
| wx wy wz    | angular velocity         |

---

### What is `default_root_state`?

This is the **initial state defined in the configuration**.

Example:

```text
position = (0,0,0)
rotation = identity
velocity = 0
```

---

### Why `.clone()` ?

```python
.clone()
```

This creates a **copy of the tensor**.

Why?

Because we want to **modify the state** without changing the original default state.

---

# 4. Add the origin positions

```python
root_state[:, :3] += origins
```

Let's break this down.

### `root_state[:, :3]`

Means:

```text
all objects
columns 0,1,2
```

Which correspond to:

```text
x y z position
```

Example:

```
root_state
[ x y z qx qy qz qw vx vy vz wx wy wz ]
```

So:

```python
:3
```

selects:

```text
[x y z]
```

---

### `+= origins`

`origins` contains the base spawn positions.

Example:

```python
origins = [
 [0.25, 0.25, 0],
 [-0.25, 0.25, 0],
 [0.25, -0.25, 0],
 [-0.25, -0.25, 0]
]
```

So each cone is moved to one of these locations.

Example:

```
Cone 1 → (0.25, 0.25, 0)
Cone 2 → (-0.25, 0.25, 0)
Cone 3 → (0.25, -0.25, 0)
Cone 4 → (-0.25, -0.25, 0)
```

# 5. Add random position offset

```python
root_state[:, :3] += math_utils.sample_cylinder(...)
```

This adds **random offsets** to the positions.

---

## What `sample_cylinder()` does

It samples random points **inside a cylinder**.

Parameters:

```python
radius=0.1
h_range=(0.25, 0.5)
```

Meaning:

| Parameter | Meaning         |
| --------- | --------------- |
| radius    | cylinder radius |
| h_range   | height range    |

So objects spawn within:

```text
radius = 10 cm
height = 25–50 cm
```

Example spawn:

```
cone
   ↓
0.3m above ground
```

Then gravity makes them **fall**.

---

### `size=cone_object.num_instances`

This tells how many random positions to generate.

Example:

```text
4 cones → 4 random samples
```

---

### `device=cone_object.device`

This ensures tensors are created on the correct device:

```text
CPU
or
GPU
```
