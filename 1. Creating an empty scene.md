I’ll explain this **in a very simple way**, focusing on the **big ideas** rather than the details. This tutorial is about **how to start Isaac Sim from a Python script and run an empty simulation**.

Think of it like **starting a game engine, creating an empty world, and running the game loop**.

---

# 1. What this tutorial is teaching

The goal is to learn how to:

1️⃣ Start the **Isaac Sim simulator**
2️⃣ Create an **empty simulation scene**
3️⃣ Run the **simulation loop**

At the end, you will have:

```
Isaac Sim running
Empty world
Simulation stepping
```

---

# 2. The two most important classes

This tutorial introduces **two main classes**.

## 1️⃣ `AppLauncher`

Purpose:

```
Start the Isaac Sim application
```

Think of it like:

```
Pressing the "start simulator" button
```

It handles things like:

* opening the simulator window
* running in headless mode
* livestream rendering
* GPU settings

---

## 2️⃣ `SimulationContext`

Purpose:

```
Control the simulation
```

It handles things like:

* physics settings
* stepping the simulation
* resetting the simulation
* camera control

Think of it like the **simulation controller**.

---

# 3. Step 1 — Launch the simulator

First we start Isaac Sim.

```python
from isaaclab.app import AppLauncher
```

Then we create a **command line parser**:

```python
parser = argparse.ArgumentParser()
```

This lets you run commands like:

```
--headless
--gpu
--livestream
```

Then we add Isaac Sim options to the parser:

```python
AppLauncher.add_app_launcher_args(parser)
```

Now we parse the command line:

```python
args_cli = parser.parse_args()
```

Finally we start the simulator:

```python
app_launcher = AppLauncher(args_cli)
simulation_app = app_launcher.app
```

Now **Isaac Sim is running**.

Important rule:

⚠️ You must start the simulator **before importing some Isaac Sim modules**, otherwise they won't work.

---

# 4. Step 2 — Import simulation modules

Now that Isaac Sim is running, we can import simulation tools.

```python
from isaaclab.sim import SimulationCfg, SimulationContext
```

These handle the **physics simulation**.

---

# 5. Step 3 — Configure the simulation

Next we define **simulation settings**.

```python
sim_cfg = SimulationCfg(dt=0.01)
```

Here:

```
dt = simulation timestep
```

Meaning:

```
simulation step = 0.01 seconds
```

So the physics updates every:

```
100 times per second
```

---

# 6. Create the simulation controller

Now we create the **SimulationContext**.

```python
sim = SimulationContext(sim_cfg)
```

This object now controls:

```
physics
timeline
stepping
rendering
```

---

# 7. Set the camera

We also define where the camera looks.

```python
sim.set_camera_view([2.5,2.5,2.5], [0,0,0])
```

Meaning:

```
Camera position → [2.5, 2.5, 2.5]
Camera looking at → [0,0,0]
```

So the camera looks at the center of the world.

---

# 8. What exists in the scene right now?

Right now we have:

```
Simulator running
Physics engine configured
Camera positioned
```

But the world is **empty**.

No:

* robots
* objects
* sensors
* lights

Those will be added in later tutorials.

---

# 9. Step 4 — Start the simulation

Before running the simulation, we must call:

```python
sim.reset()
```

This does two important things:

```
1. Starts the simulation timeline
2. Initializes physics objects
```

Important note:

```
reset() ≠ play()
```

| Method  | What it does                          |
| ------- | ------------------------------------- |
| reset() | initialize physics + start simulation |
| play()  | only starts timeline                  |

So **reset() must be called first**.

---

# 10. Step 5 — The simulation loop

Now we run the simulation.

```python
while simulation_app.is_running():
    sim.step()
```

This is the **main simulation loop**.

Every loop:

```
sim.step()
```

does:

```
1. update physics
2. update objects
3. update rendering
```

So the simulation runs continuously.

---

# 11. Stopping the simulator

When finished, we close the simulator.

```python
simulation_app.close()
```

This closes:

```
Isaac Sim window
simulation process
```
